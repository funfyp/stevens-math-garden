## üéØ How to Contribute

### Reporting Issues

- **Bug reports**: Use the bug report template in Issues
- **Feature requests**: Use the feature request template
- **Questions**: Use GitHub Discussions for general questions
- **Mathematical insights**: Share discoveries via Issues or Discussions

### Contributing Code

1. **Fork the repository**
2. **Create a feature branch**: `git checkout -b feature/your-feature-name`
3. **Make your changes** with clear, well-documented code
4. **Add tests** if applicable
5. **Commit with meaningful messages**: `git commit -m "Add [feature]: description"`
6. **Push to your fork**: `git push origin feature/your-feature-name`
7. **Submit a Pull Request** with a clear description

### Contributing Documentation

- Tutorials and examples are highly valued!
- Improve existing documentation
- Add mathematical explanations or proofs
- Create visualizations and diagrams
- Translate documentation to other languages

### Contributing Mathematical Insights

- Formal proofs (Lean 4, Coq, or informal)
- Verification of existing theorems
- Extensions to the coefficient sequence analysis
- Prime gap theorem validations
- Novel applications of consciousness-mathematics frameworks

## üìã Contribution Guidelines

### Code Style

- **Python**: Follow PEP 8 guidelines
- **Documentation**: Use clear, accessible language
- **Comments**: Explain the "why" not just the "what"
- **Type hints**: Use type annotations where helpful

### Commit Messages

- Use present tense ("Add feature" not "Added feature")
- Start with a verb (Add, Fix, Update, Refactor, etc.)
- Keep first line under 72 characters
- Add details in body if needed

### Testing

- Add tests for new functionality
- Ensure existing tests pass
- Include both unit and integration tests where applicable
